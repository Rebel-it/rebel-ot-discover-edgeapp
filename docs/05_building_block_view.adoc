== Building Block View

=== Level 1 -- Whitebox Overall System

The solution is composed of six projects: a React web application, an ASP.NET Core API, three protocol connector libraries, and a shared kernel.
The API depends only on interfaces; each connector library implements those interfaces independently, keeping the orchestration logic protocol-agnostic.

|===
| Building Block | Responsibility | Key Interface(s)

| *Rebelit.OT.Discover.EdgeApp.WebApp*
| React/TypeScript SPA that provides a step-by-step wizard for connecting to IXON Cloud, discovering OPC UA nodes, and managing variables and tags.
| — (UI client; calls the API via HTTP)

| *Rebelit.OT.Discover.EdgeApp.API*
| ASP.NET Core Web API. Hosts all controllers, services, the scraper, and the `AuthenticationFilter` that reads the IXON service account from request headers and populates `IIxonAuthenticationContext`.
| Consumes `IXON.IApiClient`, `SecureEdgePro.IApiClient`, `IUAClientFactory`, `IClientSamplerFactory`, `IScraper`

| *Rebelit.OT.Discover.EdgeApp.Connections.IXON*
| Adapts the IXON Cloud REST API v2. Builds authenticated `HttpClient` instances (service account headers), handles paging, JSON (de)serialisation, and error logging.
| Exposes `IApiClient`; registers via `AddIXONClient()`

| *Rebelit.OT.Discover.EdgeApp.Connections.OPCUA*
| Wraps the OPC Foundation .NET Standard SDK. Manages OPC UA sessions, keep-alive, reconnection, and full address-space traversal.
| Exposes `IUAClientFactory`, `IClientSamplerFactory`; registers via `AddOPCUAClient()`

| *Rebelit.OT.Discover.EdgeApp.Connections.SecureEdgePro*
| Calls the local SecureEdge Pro device REST API to retrieve device system information (serial number, firmware) needed to look up the correct IXON agent.
| Exposes `IApiClient`; registers via `AddSecureEdgeProClient()`

| *Rebelit.OT.Discover.EdgeApp.SharedKernel*
| Cross-cutting types shared by the API and all connector projects: `IIxonAuthenticationContext`, `IxonHeaders`, `ServiceAccount`, and the `Result<T>` return wrapper.
| `IIxonAuthenticationContext`, `Result<T>`
|===

=== Level 2 -- Internal Structure of Building Blocks

==== 5.2.1 Whitebox WebApp (React Frontend)

The frontend is a Vite/React SPA that guides the user through a multi-step wizard. Each page calls the backend API via a typed service layer; credentials from the login step are stored in session storage and attached as HTTP headers on every subsequent request.

        p4["sourcePage\n(data source select / create)"]

[options="header", cols="2,4"]
|===
| Component | Responsibility

| Pages (`loginPage` … `tagPage`)
| Each page corresponds to one wizard step. `loginPage` collects the IXON service account credentials and stores them via `sessionStorageService`. The remaining pages consume the typed services below, passing credentials as request headers via `httpClient`.

| `sessionStorageService`
| Persists the service account application ID, access token, company ID, agent ID, OPC UA endpoint, credentials, and data source ID in browser session storage so they survive page navigation without re-entry.

| `companyConfigurationService`
| Calls `GET /api/companyconfiguration`. Retrieves the company associated with the service account and the serial number of the SecureEdge Pro device, used to locate the corresponding IXON agent automatically.

| `plcService`
| Calls `POST /api/plc/connect`. Verifies that an OPC UA connection can be established with the provided endpoint and credentials.

| `dataSourceService`
| Calls `POST /api/ixonsettings/datasource`. Resolves or creates the OPC UA data source on the selected IXON agent.

| `scraperService`
| Calls `POST /api/scraper/variables`. Triggers the variable discovery run and returns the count of newly created variables.

| `variableService`
| Calls `GET /api/variable`. Returns all IXON data variables for the configured agent.

| `tagService`
| Calls `GET /api/tags`, `GET /api/tags/prefilled`, `POST /api/tags`, `POST /api/tags/CreateTags`, and `PUT /api/tags`. Manages tag retrieval and creation/update.

| `IxonSettingService`
| Calls `POST /api/ixonsettings`. Manages IXON settings for the configured agent.
|===

[options="header", cols="2,4"]
|===
| Component | Responsibility

| `AuthenticationFilter`
| `IActionFilter` applied globally. Reads `Api-Application` and `Api-Access-Token` from the request headers (returns `401` if absent) and writes them into the scoped `IIxonAuthenticationContext` together with the optional `Api-Company-Id`, `Api-Agent-Id`, OPC UA, and `Source-Id` headers.

| `CompanyConfigurationController`
| `GET /api/companyconfiguration`. Delegates to `ICompanyConfigurationService` to auto-detect company ID from the service account and match the corresponding IXON agent via the SecureEdge Pro serial number.

| `IxonSettingsController`
| `POST /api/ixonsettings/datasource`. Delegates to `IDataSourceResolver` to find an existing OPC UA data source by name or create a new one on the agent.

| `PlcController`
| `POST /api/plc/connect`. Attempts an OPC UA connection using the supplied credentials; returns `200 OK` on success or `400 Bad Request` on failure.

| `ScraperController`
| `POST /api/scraper/variables`. Calls `IScraper.ExecuteVariableScraperAsync`, which browses the OPC UA address space and pushes newly discovered nodes to IXON as variables. Returns the count of created variables.

| `TagsController`
| `GET`, `POST`, and `PUT` endpoints under `/api/tags`. Delegates to `ITagService` for listing existing tags, generating pre-filled tag suggestions from discovered variables, creating individual or bulk tags, and updating tag settings.

| `VariableController`
| `GET /api/variable`. Delegates to `IVariableService` to retrieve all data variables for the configured agent from IXON.

| `ICompanyConfigurationService`
| Calls `IXON.IApiClient.GetAssociatedCompanyAsync()` and `SecureEdgePro.IApiClient.GetSystemInfoAsync()`. Sets the company ID on `IIxonAuthenticationContext` and finds the matching IXON agent by serial number.

| `ITagService`
| Wraps IXON tag read and write operations. `GetPrefilledTagsAsync` generates default tag configurations for all variables that do not yet have a tag.

| `IVariableService`
| Calls `IXON.IApiClient.GetDataVariablesAsync()` and maps the results to API response DTOs.

| `IDataSourceResolver`
| Looks up an existing data source on the agent by name (slug-matched). Creates a new OPC UA data source if none is found.

| `IScraper` / `Scraper`
| Core discovery logic. Reads OPC UA node address space, then for each discovered node that is not yet present in IXON: maps the node to a `Variable` and POSTs it in batches to the IXON API.
|===

[options="header", cols="2,4"]
|===
| Component | Responsibility

| `IApiClient`
| Public interface consumed by the API services and scraper. Declares all IXON Cloud REST operations: variables, tags, data sources, agents, devices, and company lookup.

| `BaseAgent`
| Internal abstract base. On each request, reads the `ServiceAccount` (application ID + access token) and optional `CompanyId` from `IIxonAuthenticationContext` and creates a fresh `HttpClient` with the correct `Authorization: Bearer`, `Api-Application`, `Api-Company`, and `Api-Version` headers. All HTTP calls are wrapped in a Polly resilience pipeline (retry on transient errors). Logs `400 Bad Request` response bodies for debugging.

| `ApiClient`
| Concrete implementation of `IApiClient`. Builds endpoint URIs from `Configuration.BaseUrl` and delegates every call to `BaseAgent.Get<T>`, `Post<T>`, `Patch<T>`, or `Put<T>`.

| `Configuration`
| Settings record bound from `IOptionsMonitor<Configuration>`. Holds `BaseUrl` (default `https://portal.ixon.cloud`) and `Version` (default `2`). Credentials are no longer stored here -- they arrive per-request via `IIxonAuthenticationContext`.

| *Models* (`Variable`, `Tag`, `DataSource`, `DataSourceProtocol`, `Device`, `Agent`, `Company`, `Response<T>`)
| DTOs for the IXON REST API. `Response<T>` wraps the `.Data` payload. `DataSourceProtocol` carries the OPC UA protocol identifier and credential fields. `Company` and `Agent` are used during the auto-detect flow.
|===

[options="header", cols="2,4"]
|===
| Component | Responsibility

| `IUAClientFactory`
| Public interface consumed by `PlcController` and `Scraper`. Overloads for anonymous and username/password connections; returns a connected `UAClient` or `null` on failure.

| `IClientSamplerFactory`
| Public interface consumed by `Scraper`. Returns a `ClientSamples` instance ready to browse.

| `UAClientFactory`
| Constructs a `UserIdentity` from the supplied credentials (or `AnonymousIdentity` when none are provided), instantiates `UAClient`, calls `ConnectAsync`, and returns the connected client.

| `UAClient`
| Wraps an OPC UA `ISession`. Manages session establishment, certificate acceptance (`AutoAccept = true`), keep-alive (5 s), and `SessionReconnectHandler` with 1 s initial period and 15 s exponential back-off cap. Implements `IDisposable`.

| `ClientSamples`
| Address-space traversal over an `ISession`. `BrowseFullAddressSpaceAsync` recursively walks the node tree from a given root up to depth 128 and returns all `ReferenceDescription` objects.

| `ITelemetryContext` / `ConsoleTelemetry`
| Thin abstraction over `ILoggerFactory` that routes OPC UA SDK log output into the Serilog pipeline without coupling the SDK to a concrete logger.

| `ServiceCollectionExtensions`
| `AddOPCUAClient(applicationName)` registers `ApplicationInstance` (auto-accept certificates, min key size 1024) and all factories and telemetry as singletons.
|===

==== 5.2.5 Whitebox Connections.SecureEdgePro

[options="header", cols="2,4"]
|===
| Component | Responsibility

| `IApiClient`
| Public interface with a single operation: `GetSystemInfoAsync()`. Returns a `Result<SecureEdgeSystemInfo>` containing the device serial number and firmware details.

| `ApiClient`
| Uses a named `IHttpClientFactory` client (`"SecureEdgePro"`) to call `GET /api/v1/system/info` on the local SecureEdge Pro. Handles HTTP errors and network failures by returning a `Result` with an error message rather than throwing.

| `SecureEdgeSystemInfo`
| DTO containing the device serial number and other system properties returned by the SecureEdge Pro local API.

| `ServiceCollectionExtensions`
| Registers `ApiClient` as the scoped `IApiClient` and configures the named `HttpClient` with the device base address.
|===

==== 5.2.6 Whitebox SharedKernel

[options="header", cols="2,4"]
|===
| Component | Responsibility

| `IIxonAuthenticationContext` / `IxonAuthenticationContext`
| Scoped carrier for the per-request `IxonHeaders`. The `AuthenticationFilter` writes into it once per request; all connectors and services read from it throughout the same request lifetime.

| `IxonHeaders`
| Groups all per-request context: `ServiceAccount` (application ID + access token), optional `CompanyId`, `AgentId`, OPC UA endpoint (`PlcUrl`), OPC UA credentials (`PlcUsername`, `PlcPassword`), and `SourceId`.

| `ServiceAccount`
| Immutable record holding `ApiApplicationId` and `AccessToken` -- the two credentials that identify the IXON service account.

| `Result<T>`
| Generic return wrapper used by `SecureEdgePro.IApiClient` and service layer methods to convey success (`Data`) or failure (`ErrorMessage`) without throwing exceptions across layer boundaries.
|===
