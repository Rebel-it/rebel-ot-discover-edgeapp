[[section-concepts]]
== Crosscutting Concepts

This section describes the practices and patterns that apply consistently across multiple building blocks.

=== 8.1 Authentication and Credential Flow

IXON service account credentials are never stored on the device. The browser supplies them on every request as HTTP headers; the `AuthenticationFilter` validates them and makes them available to all downstream components via the scoped `IIxonAuthenticationContext`.

[options="header", cols="2,3"]
|===
| Header | Content

| `Api-Application` | IXON service account application ID (required)
| `Api-Access-Token` | IXON service account access token (required)
| `Api-Company-Id` | IXON company public ID (optional, resolved automatically)
| `Api-Agent-Id` | Target IXON agent ID (optional, resolved automatically)
| `OpcUa-Server-Address` | OPC UA endpoint URL (optional)
| `OpcUa-Username` | OPC UA username (optional)
| `OpcUa-Password` | OPC UA password (optional)
| `Source-Id` | OPC UA data source ID on the agent (optional)
|===

The only environment variable relevant to the backend at runtime is `LOG_LEVEL` (default: `Information`) and `SECUREEDGEPRO_BaseAddress`, which is auto-detected from the container's default gateway if not set explicitly.

=== 8.2 Logging

All .NET components write structured log messages through `ILogger<T>`. Serilog is configured once in `Program.cs` as the backing implementation.

Log levels in use:

* `Information` — request handling, skip decisions, cycle progress
* `Error` — failed OPC UA connections, HTTP `400` response bodies, unexpected exceptions

No credentials, OPC UA node values, or full response bodies are ever logged. `BaseAgent` logs only the body of `400 Bad Request` responses to support debugging of API rejections.
The log level can be adjusted at deploy time via the `LOG_LEVEL` environment variable without rebuilding the image.

=== 8.3 Idempotency

The variable discovery run is fully idempotent. Before posting any new variable, the scraper fetches all existing variables from IXON and filters the OPC UA browse results against that set.

[mermaid]
....
flowchart TD
    start["ExecuteVariableScraperAsync begins"]
    browse["Browse OPC UA address space"]
    fetch["GET all existing Variables from IXON"]
    check{{"Address already\nin IXON?"}}
    skip["Skip node"]
    post["POST Variable to IXON (batched)"]

    start --> browse --> fetch --> check
    check -->|"yes"| skip
    check -->|"no"| post
    post --> check
    skip --> check
....

Re-running the scraper after a partial failure never creates duplicate variables. The overhead is one paginated GET request per run, capped at 4,000 entries.

=== 8.4 Error Handling

[options="header", cols="2,2,4"]
|===
| Layer | Location | Behaviour

| HTTP response errors
| `BaseAgent`
| `400 Bad Request` responses log the response body before propagating via `EnsureSuccessStatusCode()`. Other non-2xx responses throw `HttpRequestException`, which propagates to the controller and results in a `500` to the client.

| Service-layer results
| `CompanyConfigurationService`, `SecureEdgePro.ApiClient`
| Return `Result<T>` rather than throwing, allowing the controller to return a structured `400 Bad Request` with an error message instead of an unhandled exception.

| OPC UA connection failure
| `UAClientFactory`
| Returns `null` when a connection cannot be established. The caller (`PlcController`, `Scraper`) checks for `null` and returns a descriptive error response to the frontend.
|===

=== 8.5 Dependency Injection and Modular Composition

`Program.cs` is the sole composition root. Each project exposes one `IServiceCollection` extension method that hides all internal registrations behind a clean surface.

[options="header", cols="2,3"]
|===
| Extension Method | Registers

| `AddIXONClient()` | `IApiClient` (scoped), `IOptionsMonitor<Configuration>`, `IIxonAuthenticationContext`
| `AddOPCUAClient(appName)` | `IUAClientFactory`, `IClientSamplerFactory`, `ITelemetryContext`, `ApplicationInstance`
| `AddSecureEdgeProClient()` | `SecureEdgePro.IApiClient`, named `HttpClient` with device base address
|===

The API project depends only on the interface contracts from each connector project. The concrete implementations are never referenced directly, which makes them independently testable and replaceable.

=== 8.6 Domain Data Model

[mermaid]
....
classDiagram
    class Variable {
        +string PublicId
        +string Name
        +string Address
        +string DataType
        +int BitWidth
        +Source Source
    }

    class Tag {
        +string PublicId
        +string Name
        +string RetentionPolicy
        +string LogEvent
        +string LoggingInterval
        +VariableRef Variable
    }

    class Source {
        +string PublicId
    }

    class VariableRef {
        +string PublicId
    }

    Variable "1" --> "1" Source : belongs to
    Tag "1" --> "1" VariableRef : references
    Variable "1" -- "0..1" Tag : paired with
....

Each OPC UA node is mapped to one `Variable` (node address and data type) and optionally one `Tag` (logging configuration). The `Source` identifies the OPC UA data source on the IXON agent the variable belongs to. Variables are created by the scraper; tags are reviewed and created separately by the user through the wizard.
