= Getting Started
:toc:
To install this EdgeApp onto a SecureEdge Pro, you will need to build the EdgeApp image and push it to a container registry. You can then deploy the EdgeApp through the internal SecureEdge Pro interface.


== Prerequisites
- link:https://docs.docker.com/get-docker/[Docker] with link:https://docs.docker.com/buildx/working-with-buildx/[BuildKit / `docker buildx`] enabled.
- An IXON SecureEdge Pro device accessible on your local network (or with VPN connection through the portal).
- An IXON Cloud account with the *API Integrations* module enabled.

== Building the EdgeApp Image
The script link:docker/push_ixon.sh[`docker/push_ixon.sh`] builds a `linux/arm64/v8` image and pushes it directly to the container registry on the SecureEdge Pro. Run it from the `docker/` directory:

[source,bash]
----
cd docker
./push_ixon.sh
----

The script will prompt for nine values:

[cols="1,2,2", options="header"]
|===
| Prompt | Description | How to obtain

| *SecureEdge IP address*
| The LAN IP address of your IXON SecureEdge Pro (e.g. `172.27.21.1`).
| Check your network configuration or the IXON portal under the device's network settings.

| *IXON Application ID*
| Identifies the client application calling the IXON API.
| Contact your IXON Sales representative or account manager to request an Application ID.

| *IXON Company ID*
| The `publicId` of your IXON company.
| Retrieve it via the IXON API link:https://developer.ixon.cloud/reference/get_companies[`CompanyList`] endpoint, or find it in the IXON portal URL when you navigate to your company settings.

| *IXON Bearer Token*
| A temporary API access token used to authenticate all API calls.
| Generate one via the IXON API link:https://developer.ixon.cloud/reference/post_access-tokens[`AccessTokenList`] endpoint (see below).

| *IXON Agent ID*
| The `publicId` of the IXON agent (machine) to associate discovered data with.
| Find it in the IXON portal under the device's agent settings, or via the IXON API.

| *IXON Source Public ID*
| The `publicId` of the data source on the IXON agent.
| Find it in the IXON portal under the agent's data sources, or via the IXON API.

| *OPC UA Server Address*
| The OPC UA endpoint URL of the server to connect to (e.g. `opc.tcp://172.27.21.3:4840`).
| Check the configuration of your OPC UA server or PLC.

| *OPC UA Username*
| Username for authenticating with the OPC UA server.
| Check the credentials configured on your OPC UA server or PLC.

| *OPC UA Password*
| Password for authenticating with the OPC UA server.
| Check the credentials configured on your OPC UA server or PLC.
|===

The credentials are baked into the Docker image as environment variables at build time, so no secrets need to be managed on the device after deployment.

=== Generating an IXON Bearer Token
A bearer token is obtained by calling the `AccessTokenList` endpoint with your IXON email and password using HTTP Basic Auth. The token expires after the duration you specify (`expiresIn`, in seconds; maximum 60 days / 5 184 000 seconds).

[source,bash]
----
# 1. Base64-encode your credentials (email::password)
CIPHER=$(echo -n 'your@email.com::YourPassword' | base64)

# 2. Request a bearer token (here valid for 24 hours)
curl --request POST \
    --url 'https://portal.ixon.cloud/api/access-tokens?fields=secretId' \
    --header 'Api-Version: 2' \
    --header "Api-Application: YOUR_APPLICATION_ID" \
    --header 'Content-Type: application/json' \
    --header "Authorization: Basic $CIPHER" \
    --data '{"expiresIn": 86400}'
----

The `secretId` value in the response is your bearer token. Enter only the token value when prompted by `push_ixon.sh` (without the `Bearer` prefix).

[NOTE]
====
If your account uses Two-Factor Authentication, encode your credentials as `email:otp:password` instead.
====

== Deploying the EdgeApp
After the image is built and pushed to the SecureEdge Pro, you can deploy it through the internal interface:

. Log in to the SecureEdge Pro's web interface (credentials are located on the device).
. Navigate to the *Docker* section and click *Add* button of the left of the `rebel-ot-discover-edgeapp` image.

image::docs/assets/sourceedge_docker_image.png[]

[start="3"]
. Add a container name and click *Create* to deploy the EdgeApp.

image::docs/assets/sourceedge_docker_container.png[]

[start="4"]
. Press the *Run* (1) button to start the container. You can view the logs by clicking the *Logs* (2) button.

image::docs/assets/sourceedge_docker_run.png[]
