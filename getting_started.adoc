= Getting Started
:toc:
This guide walks you through installing the *Rebelit OT Discover Edge App* on an IXON SecureEdge Pro and completing your first variable discovery run.

The application consists of two Docker containers deployed on the SecureEdge Pro:

* A *backend* (ASP.NET Core API) that talks to IXON Cloud, the OPC UA server, and the SecureEdge Pro device API.
* A *frontend* (React web app served by nginx) accessible in your browser at `http://<device-ip>:3000`.

== Prerequisites

* link:https://docs.docker.com/get-docker/[Docker] with link:https://docs.docker.com/buildx/working-with-buildx/[BuildKit / `docker buildx`] enabled.
* An IXON SecureEdge Pro device accessible on your local network (or via VPN through the IXON portal).
* An IXON Cloud account with the *API Integrations* module enabled.
* An IXON service account with an *Application ID* and *Access Token* (see <<service-account,Setting up an IXON Service Account>> below).

== Installation

Run the install script from the `docker/` directory. It builds both images, pushes them to the SecureEdge Pro's local registry, and creates and starts the containers — all in one step.

[source,bash]
----
cd docker
./install_discover_edgeapp.sh
----

The script will prompt for:

[cols="1,3", options="header"]
|===
| Prompt | Description

| *SecureEdge IP address*
| The LAN IP of your SecureEdge Pro (e.g. `172.27.21.1`). Check your network configuration or the IXON portal under the device's network settings.

| *Username SecureEdge Pro*
| Admin username for the SecureEdge Pro web interface (default: `admin`). Found on the label on the device.

| *Password SecureEdge Pro*
| Admin password for the SecureEdge Pro web interface. Found on the label on the device.

| *Log level* _(optional)_
| Minimum log level for the backend container output. Accepted values: `Verbose`, `Debug`, `Information`, `Warning`, `Error`, `Fatal`. Defaults to `Information`.
|===

Once complete, the script prints:

----
=== Installation complete ===
Please visit the application on your Secure Edge Pro: http://<device-ip>:3000
----

[[insecure-registry]]
[NOTE]
====
The SecureEdge Pro registry (`<device-ip>:5000`) must be configured as an *insecure registry* in Docker Desktop before running the script. +
Add it via *Docker Desktop → Settings → Docker Engine*:
[source,json]
----
{ "insecure-registries": ["<device-ip>:5000"] }
----
Then restart Docker and re-run the script.
====

== End-User Deployment (Installation Wizard)

The recommended way to install or uninstall the Edge App is through the *Installation Wizard* — a cross-platform console application that automatically downloads the latest release from GitHub and runs the appropriate deployment scripts for your operating system.

=== Prerequisites

* link:https://dotnet.microsoft.com/en-us/download/dotnet/10.0[.NET 10 Runtime] installed on your machine.
* A GitHub personal access token with read access to the repository (required until the repository is made public).
* The same <<insecure-registry,insecure registry>> prerequisite as described above.

=== Running the Wizard

Run the wizard executable from a terminal:

[source,bash]
----
# Linux / macOS
./Rebelit.OT.InstallationWizard

# Windows (PowerShell)
.\Rebelit.OT.InstallationWizard.exe
----

The wizard will guide you through the following prompts:

[cols="1,3", options="header"]
|===
| Prompt | Description

| *Action*
| Choose `[1] Install` to deploy the Edge App or `[2] Uninstall` to remove it.

| *SecureEdge Pro IP address*
| The LAN IP of your SecureEdge Pro (e.g. `192.168.140.1`).

| *SecureEdge Pro admin username*
| Admin username for the SecureEdge Pro web interface (default: `admin`).

| *SecureEdge Pro admin password*
| Admin password for the SecureEdge Pro web interface.

| *GitHub access token*
| Personal access token used to download the latest release from GitHub.
|===

The wizard then downloads and extracts the latest release automatically, and runs the deployment. Once complete, it opens your browser at `http://<device-ip>:3000`.

== Using the Wizard

Open `http://<device-ip>:3000` in your browser. The wizard guides you through the following steps:

. *Login* — Enter your IXON service account Application ID and Access Token.
. *Company & Agent detection* — The app automatically retrieves your IXON company and matches the SecureEdge Pro device to the correct IXON agent using its serial number.
. *OPC UA connection* — Enter the OPC UA server address and credentials. The app verifies the connection before you proceed.
. *Data source* — Select an existing OPC UA data source on the agent, or let the app create one automatically.
. *Variable discovery* — Trigger the discovery run. The app browses the OPC UA address space and registers all new nodes as IXON data variables.
. *Tags* — Review the discovered variables and create data tags with your preferred logging configuration.

== Setting up an IXON Service Account

The wizard requires an IXON service account *Application ID* and *Access Token*. These are different from your personal IXON login.

=== Application ID
Contact your IXON account manager or sales representative to request an API Application ID for your integration.

=== Access Token
Generate an access token by calling the IXON `AccessTokenList` endpoint with your IXON credentials using HTTP Basic Auth:

[source,bash]
----
# 1. Base64-encode your credentials (email::password)
CIPHER=$(echo -n 'your@email.com::YourPassword' | base64)

# 2. Request an access token (here valid for 60 days)
curl --request POST \
    --url 'https://portal.ixon.cloud/api/access-tokens?fields=secretId' \
    --header 'Api-Version: 2' \
    --header "Api-Application: YOUR_APPLICATION_ID" \
    --header 'Content-Type: application/json' \
    --header "Authorization: Basic $CIPHER" \
    --data '{"expiresIn": 5184000}'
----

The `secretId` value in the response is your access token. Enter it in the wizard login step.

[NOTE]
====
If your account uses Two-Factor Authentication, encode your credentials as `email:otp:password` instead.
====
